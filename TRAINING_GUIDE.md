# Fine-Tuning Guide: Gemma-2-9b-it on Synthetic CVE Dataset

This guide explains how to use the `src/cve_synth/train.py` script to fine-tune a student Gemma-2-9b-it model on your synthetic CVE dataset using Unsloth and LoRA.

## Overview

The training pipeline:
1. **Loads** your `data/dataset.jsonl` (generated by the pipeline)
2. **Formats** each record into Gemma-2 chat templates with Chain-of-Thought structure
3. **Initializes** Gemma-2-9b-it with 4-bit quantization and LoRA adapters (Unsloth-optimized)
4. **Trains** using SFTTrainer with gradient accumulation for 16GB VRAM cards (P100, RTX 2070, etc.)
5. **Saves** the final fine-tuned model and LoRA adapters

## Hardware Requirements

- **GPU**: 16GB VRAM (RTX 2070, RTX 2080, A100 40GB slice, Kaggle P100, etc.)
- **RAM**: ≥32GB (system memory)
- **Storage**: ≥50GB (for model checkpoints and final weights)

### Advanced Configuration (16GB VRAM Tuning)

For maximum memory efficiency on a 16GB GPU:

```bash
python -m cve_synth.train \
    --dataset-path data/dataset.jsonl \
    --output-dir ./output/train_results \
    --num-train-epochs 3 \
    --batch-size 1 \
    --gradient-accumulation-steps 8 \
    --learning-rate 2e-4 \
    --warmup-ratio 0.05 \
    --logging-steps 5 \
    --save-steps 250
```

**Parameter explanations:**
- `--batch-size 1`: Per-GPU batch size (aggressive memory constraint)
- `--gradient-accumulation-steps 8`: Accumulate gradients over 8 steps → effective batch size of 8
- `--num-train-epochs 3`: Train for 3 full passes over the dataset
- `--learning-rate 2e-4`: LoRA-appropriate learning rate (lower than full fine-tuning)
- `--warmup-ratio 0.05`: Warm up for 5% of training steps
- `--logging-steps 5`: Log metrics every 5 steps (verbose)
- `--save-steps 250`: Save checkpoint every 250 steps

## Training Configuration Details

### Model Architecture
- **Base Model**: `gemma-2-9b-it` (9 billion parameters, instruction-tuned)
- **Quantization**: 4-bit (bitsandbytes) → ~2.25 GB weights
- **Total Model Size**: ~4 GB (4-bit weights + activations + optimizer state)

### LoRA Configuration
- **Rank (r)**: 16 (balance between expressiveness and memory)
- **Alpha**: 16 (scaling factor, same as rank for balanced learning)
- **Dropout**: 0.0 (no dropout on LoRA layers to preserve information)
- **Target Modules**: `q_proj`, `k_proj`, `v_proj`, `o_proj`, `gate_proj`, `up_proj`, `down_proj`
- **Trainable Parameters**: ~15-20 million (0.2% of base model)

### Dataset Formatting (Chain-of-Thought)

Each record is formatted as:

```
<start_of_turn>user
Analyze the following security vulnerability and provide structured reasoning:

CVE ID: CVE-2026-XXXXX
Title: [vulnerability title]
Summary: [vulnerability summary]

Please provide:
1. Root cause analysis
2. Step-by-step reasoning (chain of thought)
3. Vulnerability classification tags
4. Assembly-level fix recommendations
<end_of_turn>
<start_of_turn>model
**Root Cause Analysis:**
[root cause text from dataset]

**Chain of Thought Reasoning:**
1. [reasoning step 1]
2. [reasoning step 2]
3. [reasoning step 3]
...

**Vulnerability Classification:**
Tags: #LogicError, #MemoryCorruption, ...

**Assembly-Level Fix:**
[assembly fix from dataset]
<end_of_turn>
```

This format:
- Explicitly structures the model's output (root cause, reasoning chain, tags, fix)
- Enforces multi-step reasoning (chain-of-thought)
- Reuses your teacher model's annotations (reasoning_chain, tags, assembly_fix)

### Training Hyperparameters

- **Optimizer**: AdamW 8-bit (memory-efficient)
- **Learning Rate**: 2e-4 (standard for LoRA)
- **Warmup**: 3% of total steps (gradual learning rate increase)
- **Weight Decay**: 0.0 (no regularization, as is standard for LoRA)
- **Gradient Checkpointing**: Unsloth-optimized (reduced activation memory)
- **Precision**: mixed-precision training (fp16 or bfloat16)

## Using the Fine-Tuned Model

### Load for Inference

```python
from transformers import AutoTokenizer, AutoModelForCausalLM
from peft import PeftModel

# Load base model
base_model = AutoModelForCausalLM.from_pretrained(
    "unsloth/gemma-2-9b-it",
    device_map="auto",
    load_in_4bit=True,
)

# Load LoRA adapters
model = PeftModel.from_pretrained(
    base_model,
    "./output/train_results/final_model",
)

tokenizer = AutoTokenizer.from_pretrained("./output/train_results/final_model")

# Use for inference
prompt = "<start_of_turn>user\nAnalyze CVE-2024-1234...\n<end_of_turn>\n<start_of_turn>model\n"
inputs = tokenizer(prompt, return_tensors="pt").to("cuda")
outputs = model.generate(**inputs, max_new_tokens=512)
print(tokenizer.decode(outputs[0], skip_special_tokens=True))
```

### Merge LoRA Weights (Optional)

To merge LoRA adapters into the base model for deployment:

```python
from peft import PeftModel
from transformers import AutoModelForCausalLM

base_model = AutoModelForCausalLM.from_pretrained(
    "unsloth/gemma-2-9b-it",
    device_map="auto",
)

model = PeftModel.from_pretrained(base_model, "./output/train_results/final_model")
merged_model = model.merge_and_unload()

# Save the merged model
merged_model.save_pretrained("./merged_gemma2_cve_synth")
```

## Troubleshooting

### Out of Memory (OOM)

If you encounter CUDA out-of-memory errors:

1. **Reduce batch size:**
   ```bash
   python -m cve_synth.train --batch-size 1 --gradient-accumulation-steps 16
   ```

2. **Increase gradient accumulation:**
   ```bash
   python -m cve_synth.train --batch-size 1 --gradient-accumulation-steps 16
   ```

3. **Enable CPU offloading (extreme):**
   Edit `train.py` and set `device_map="auto"` with CPU fallback.

### Slow Training

- **Reduce logging/saving frequency:**
  ```bash
  python -m cve_synth.train --logging-steps 50 --save-steps 2000
  ```

- **Use fewer epochs (validation first):**
  ```bash
  python -m cve_synth.train --num-train-epochs 1
  ```

### Poor Model Quality

- **Increase training data:** Generate more synthetic CVEs with the pipeline.
- **Train longer:** Increase `--num-train-epochs` (e.g., 5-10).
- **Adjust learning rate:** Try `--learning-rate 5e-4` or `--learning-rate 1e-4`.
- **Validate on test set:** Save some records separately to evaluate model improvements.
